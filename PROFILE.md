# Listings API

A local-marketing / local-SEO API providing white-label infrastructure: one write fans a business location out to major publishers (Google Business Profile, Apple Maps, Bing Places, Yelp, Facebook, Tripadvisor, etc.), plus review aggregation/response, multi-channel posting, and per-publisher analytics. Exposes a REST API, hosted MCP server, and typed SDKs. The REST API is a facade over Synup's federated GraphQL services.

## APIs

- **Listings API v4** — REST API for local marketing — locations, listings, connected accounts, reviews, posts, analytics, and supporting endpoints. 56 paths under /api/v4/. Auth via API key header (Authorization: API <key>). Also offers a hosted MCP server and an llms.txt. (`https://listingsapi.com/api/v4`)

> Three publicly consumable surfaces confirmed live: REST API with public OpenAPI 3.1 contract, hosted MCP server (https://listingsapi.com/mcp, streamable HTTP, same API-key auth), and llms.txt (plus llms-full.txt). MCP tools include docs search, OpenAPI introspection, and dev-portal control-plane tools. SDKs: Python (pip install listingsapi) and Node/TypeScript (npm install listingsapi-js). REST is a facade over Synup's internal GraphQL — no public GraphQL SDL exposed. Webhooks (review.created, listing.submission) documented in prose only, no AsyncAPI file. No published agent skills.
